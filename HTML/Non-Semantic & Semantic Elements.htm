Semantic Elements

Non-Semantic & Semantic Elements



Non-Semantic Elements

Elements that have No clear depiction of its content, it is typically an element with a generic tag name that can contain anything.

Note: There are many Non-Semantic Elements (change the "()" into "<>")
Tag Name Description
(span) marks up a part of text
(i) italicizes a part of a text
(b) bold a part of a text
(div) used as a container of elements

Class & ID Attribute

  • Are used to identify elements.
  • Can be applied to any elemets.
  • Are used to label non-semantic elements.
  • Used in styling or modifying elements.

ID (identification)

Used to identify a single element in our HTML Document, this can be used to change its design or modify and retrieve the element's value

                    (div id="main-container")
                        (p)Lorem Ipsum Dolor Sit Amet(/p)
                    (/div)

PS: Same IDs should NOT be used on multiple elements

Class

Used to identify multiple elements in our HTML Document, particularly repetitive elements that may also be used to change its design or modify and retrieve the elemen's values.

                    (div class="comment")
                        (p)This is Comment 1(/p)
                    (/div)

                    (div class="comment")
                        (p)This is Comment 2(/p)
                    (/div)

PS: Classes can be used on multiple elements.


Semantic Elements

Elements that clearly describes what it contains, to both the developer and the browser. These are elements which have specific tag names.

It does not change any styling on the actual document. It is used to structure the website in a cleaner way.

Note: These elements clearly defines what their "content" is about. (change the "()" into "<>")
Tag Name Description
(nav) defines a Navigation
(header) defines a Header
(footer) defines a Footer
(section) defines a Section
(article) defines an Article
(aside) defines a content on the side
